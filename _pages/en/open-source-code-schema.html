Metadata Schema for Open Source Code

The metadata schema for Open Source Code.

Data files are hosted on GitHub.

Version 1.0

Mandatory fields are marked with "mandatory".

We've created an example file with only mandatory fields you can use to get started.

  • schemaVersion mandatory [string]

    The version of the ORE metadata schema for open source code.

  • adminCode mandatory [string]

    Administration code from this data file

  • releases mandatory [array]

    Contains objects representing each project or source code available.

    • contact mandatory [array]
      • URL [array]

        French and English URL that can be used to reach the point of contact.

        • en [string]
        • fr [string]
      • email mandatory [string]

        An email address to contact the project.

      • name [string]

        The name of a contact or team for the project.

      • phone [string]

        The phone number to contact the project.

    • date mandatory [array]
      • created mandatory [string]

        The date the project was originally created, in YYYY-MM-DD or ISO 8601 format.

      • lastModified [string]

        The date the project was modified, in YYYY-MM-DD or ISO 8601 format.

      • metadataLastUpdated mandatory [string]

        The date the metadata of the project was last updated, in YYYY-MM-DD or ISO 8601 format.

    • description mandatory [array]

      French and English description of the project or source code.

      • en mandatory [string]
      • fr mandatory [string]
    • downloadURL [array]

      French and English URL where a distribution of the project can be found.

      • en [string]
      • fr [string]
    • homepageURL [array]

      French and English URL of the project homepage.

      • en [string]
      • fr [string]
    • languages [array]

      A list of strings with the names of the programming languages used in the project.

    • name mandatory [array]

      French and English name of the project or source code.

      • en mandatory [string]
      • fr mandatory [string]
    • organization [array]

      French and English name of organization or component within the department or agency to which the project belongs.

      • en [string]
      • fr [string]
    • partners [array]

      An array of objects for information about groups partnering on the project.

      • URL [array]

        French and English URL to a website that can be used to reach the partner.

        • en [string]
        • fr [string]
      • email [string]

        The email address for the point of contact of the partner.

      • name [array]

        French and English name of the partner.

        • en [string]
        • fr [string]
    • licenses mandatory [array]

      An array of objects containing license details.

      • URL mandatory [array]

        French and English URL of the license.

        • en mandatory [string]
        • fr mandatory [string]
      • spdxID mandatory [string]

        SPDX License identifier - https://spdx.org/licenses/

    • repositoryURL mandatory [array]

      French and English URL of the public repository of the project.

      • en mandatory [string]
      • fr mandatory [string]
    • relatedCode [array]

      An array of source code, libraries, frameworks, APIs, platforms or other software used in this project.

      • URL [array]

        French and English URL where the software can be found.

        • en [string]
        • fr [string]
      • name [array]

        French and English name of the software used in this project.

        • en [string]
        • fr [string]
    • status [string]

      The development status of the project. For example, Production, Beta, ..

    • tags mandatory [array]

      French and English keywords that will be helpful in discovering and searching for the project.

      • en mandatory [array] A list of English keywords
      • fr mandatory [array] A list of French keywords
    • vcs [string]

      A lowercase string with the name of the version control system that is being used for the project. For example, git, svn, ..

    • version [string]

      The latest version for this project. For example, 1.0.0
