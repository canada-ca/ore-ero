Schéma de métadonnées pour code source ouvert

Le schéma de métadonnées pour la section Code source ouvert.

Les données utilisées pour générer cette liste est hébergé sur GitHub.

Les champs obligatoires sont marqués en bleu foncé.

  • Nom du champ

    Description

  • schemaVersion

    [string] La version du schéma de données ORE pour le code utilisé. Par exemple, 1.0

  • adminCode

    [string] Le code d'administration - https://github.com/canada-ca/ore-ero/blob/master/_data/administrations

  • releases

    [array] Contient des objets représentant chaque version de code source mise à disposition.

  • contact
    URL
    en

    [string] ~

    fr

    [string] L'URL d'un site Web pouvant etre utilisé pour atteindre le contact.

    email

    [string] L'adresse courriel pour contacter la "release".

    name

    [string] Le nom d'un contact ou d'un département pour la "release".

    phone

    [string] Le numéro de téléphone pour contacter une "release".

  • date
    created

    [string] La date de la "release" lorsqu'elle a été crée, au format AAAA-MM-JJ ou ISO 8601.

    lastModified

    [string] La date de la "release" lorsqu'elle a été modifiée, au format AAAa-MM-JJ ou ISO 8601.

    metaLastUpdated

    [string] La date de la dernière fois les "metadata" ont été mises à jour, au format AAAA-MM-JJ ou ISO 8601.

  • description
    en

    [string] ~

    fr

    [string] Description en une ou deux phrases de "release".

  • downloadURL
    en

    [string] ~

    fr

    [string] L'URL où une distribution de la "release" peut être trouvée.

  • homepageURL
    en

    [string] ~

    fr

    [string] L'URL de la page d'accueil de "release" publique.

  • languages

    [array] Une lsite de chaîne avec les noms de langages de programmation utilisés dans la version. [lang1, lang2]

  • name
    en

    [string] ~

    fr

    [string] Le nom de la "release" en français.

  • organization
    en

    [string] ~

    fr

    [string] L'organization ou la composante du département de l'agence à laquelle la release appartient. Par exemple, "CDS" ou "Gouvernement ouvert".

  • partner

    [array] Un tableau d'objets comprenant un acronyme pour chaque agence partenaire de la "release" et l'adresse courriel de cette agence.

    URL
    en

    [string] ~

    fr

    [string] L'URL d'un site Web qui peut être utilisé pour atteindre le partenaire.

    email

    [string] L'adresse courriel du contact de l'agence partenaire.

    name

    [string] Le nom du partenaire de contact.

  • licenses

    [array of objects] Un objet contenant les détails de la license.

    URL
    en

    [string] ~

    fr

    [string] L'URL de la license de "release"

    spdxID

    [string] Identifiant de licence de SPDX - https://spdx.org/licenses/.

  • repositoryURL
    en

    [string] ~

    fr

    [string] L'URL du répertoire de la "release" publique pour les répertoires sources ouverts.

  • relatedCode

    [array] Un ensemble de code source, de bibliothèques, de frameworks, d'API, de plateformes ou d'autres logiciels utilisés dans cette "release".

    Url
    en

    [string] ~

    fr

    [string] L'URL où le logiciel peut être trouvé.

    name
    en

    [string] ~

    fr

    [string] Le nom du logiciel utilisé dans cette "release".

  • status

    [string] Le statut de développement de la version. Par exemple, Production, Beta, ..

  • tags

    [array] Un ensemble de mots clés qui seront utiles pour la decouverte et la recherche de la "release".

    en

    [tag1-en, tag2-en]

    fr

    [tag1-fr, tag2-fr]

  • vcs

    [string] Une chaîne en miniscule contenant le nom de la version du système de contrôle qui est utilisée pour la "release". Par exemple, git, svn, ..

  • version

    [string] La version de cette "release". Par exemple, 1.0.0
